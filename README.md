# 智识库（zhishiku）产品方案

## 1. 产品目标
打造一个“**个人知识库 + 大模型写作助手**”产品，满足你提出的 3 个核心能力：

1. **关键词检索文章**：输入关键词，快速定位到具体文章。
2. **语义检索片段**：输入一段大意，找到相关文章及对应段落/片段。
3. **主题自动成文**：输入主题（如“AI 是否会取代人类”），系统优先基于知识库内容，并补充公开网络信息，自动写出结构化文章。

---

## 2. 核心能力设计

### 2.1 关键词检索（精确/布尔/过滤）
- 支持：标题、标签、作者、发布时间、正文全文检索。
- 支持：AND/OR、短语匹配、时间过滤、来源过滤。
- 输出：
  - 命中文章列表（标题 + 摘要 + 相关度分）
  - 命中位置高亮（关键词在文章中的段落位置）

### 2.2 语义检索（按“意思”找内容）
- 上传文章后自动进行：
  - 文本清洗
  - 分段切片（chunk）
  - 向量化（embedding）
  - 入向量库索引
- 查询时：
  - 将用户描述转为向量
  - 在向量库中召回 Top-K 片段
  - 重排序（rerank）提升准确率
- 输出：
  - 相关文章
  - 每篇文章对应最相关片段
  - 片段出处（文章标题、段落编号、时间戳）

### 2.3 主题自动写作（知识库优先 + 联网补充）
- 输入一个主题后，流程如下：
  1. 从知识库召回相关内容（优先）
  2. 若知识库信息不足，触发公开网络搜索补充
  3. 大模型生成“提纲 → 初稿 → 润色稿”
  4. 在文末附参考来源与引用标注
- 可选写作风格：
  - 理性分析 / 科普解释 / 辩论型 / 公众号风格
- 输出要求：
  - 观点清晰
  - 有论据支撑
  - 引用可追溯

## 3. 系统架构（建议）

```text
[前端 Web/APP]
     |
[API 网关]
     |
+-----------------------------+
|  检索编排层 Retrieval Orchestrator |
+-----------------------------+
   |             |              |
[全文检索]     [向量检索]      [联网搜索]
(ES/OpenSearch) (Milvus/pgvector) (Bing/SerpAPI)
   \             |             /
    +------[重排序 Reranker]----+
                 |
            [LLM 生成层]
                 |
          [结果+引用返回]
```

### 组件说明
- **文档存储**：PostgreSQL / MongoDB
- **全文检索**：Elasticsearch / OpenSearch
- **向量数据库**：Milvus / Weaviate / pgvector
- **模型服务**：OpenAI 兼容接口（embedding + chat）
- **任务队列**：Celery / RabbitMQ（用于批量入库、重建索引）

---

## 4. 数据流程设计

### 4.1 入库流程（离线）
1. 用户上传文章（Markdown/PDF/Word/网页剪藏）
2. 内容解析 + 元数据抽取（标题、标签、时间）
3. 分段切片（建议 300–800 tokens）
4. embedding 向量化
5. 同时写入：
   - 文档库（原文）
   - 全文索引（关键词检索）
   - 向量索引（语义检索）

### 4.2 查询流程（在线）
- **关键词模式**：用户 Query → 全文检索 → 返回文章结果
- **语义模式**：用户 Query → 向量召回 + 重排序 → 返回片段
- **写作模式**：主题 → 检索编排层融合多源证据 → LLM 成文 → 返回带引用文章

---

## 5. 检索与生成策略（关键）

### 5.1 混合检索
采用 `Hybrid Search = BM25 + Vector Search + Rerank`：
- BM25 保证“关键词强匹配”
- 向量检索保证“语义相关”
- Rerank 保证最终排序质量

### 5.2 引用可信度控制
- 每段生成内容绑定至少一个证据片段（知识库或网络）
- 明确区分：
  - 来自知识库的观点
  - 来自公开网络的信息
- 对网络来源做白名单和时效性限制（如近 1 年）

### 5.3 幻觉防护
- 设置“仅基于证据回答”模式（无证据则提示信息不足）
- 对低置信度段落加提示
- 提供“一键查看证据”侧栏

## 7. API 草案

### 7.1 关键词检索
`POST /api/search/keyword`

```json
{
  "query": "向量数据库",
  "filters": {
    "tags": ["AI"],
    "date_from": "2025-01-01"
  },
  "top_k": 10
}
```

### 7.2 语义检索
`POST /api/search/semantic`

```json
{
  "query": "我想找讨论 AI 幻觉和引用可信度的文章",
  "top_k": 8,
  "return_chunks": true
}
```

### 7.3 主题写作
`POST /api/write/topic`

```json
{
  "topic": "AI 是否会取代人类",
  "style": "analysis",
  "length": "1500",
  "use_web": true,
  "kb_priority": true
}
```

## 8. 评估指标（上线必须监控）
- 检索准确率：Recall@K, nDCG@K
- 生成质量：引用覆盖率、事实一致性评分
- 体验指标：首字响应时间（TTFT）、整文生成时长
- 业务指标：日活、检索成功率、写作采纳率

---

## 9. 技术选型建议（务实版）
- 后端：Python + FastAPI
- 全文检索：OpenSearch
- 向量：pgvector（早期成本低，便于一体化部署）
- 模型：
  - Embedding 模型（中文语义检索效果优先）
  - 通用对话模型（写作）
- 前端：Next.js + Tailwind（快速搭建工作台）
